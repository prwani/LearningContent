6.1 Structured Output ‚Äî Guaranteed JSON

Both providers support forcing the model to output valid JSON matching a schema.

OpenAI 
// Chat Completions
{
  "response_format": {
    "type": "json_schema",
    "json_schema": {
      "name": "weather_report",
      "schema": {
        "type": "object",
        "properties": {
          "city": {"type": "string"},
          "temp_f": {"type": "number"},
          "condition": {"type": "string"}
        },
        "required": ["city","temp_f","condition"]
      }
    }
  }
}

// Responses API: same, via text.format
{
  "text": {
    "format": {
      "type": "json_schema",
      "schema": { ... }
    }
  }
}
Anthropic 
// Use a tool with exact schema
// to force structured output
{
  "tools": [{
    "name": "weather_report",
    "description": "Output weather data",
    "input_schema": {
      "type": "object",
      "properties": {
        "city": {"type": "string"},
        "temp_f": {"type": "number"},
        "condition": {"type": "string"}
      },
      "required": ["city","temp_f","condition"]
    }
  }],
  "tool_choice": {
    "type": "tool",
    "name": "weather_report"
  }
}

// Force model to use this specific tool
// Output comes as tool_use block
Different approaches: OpenAI has a dedicated response_format for JSON output. Anthropic achieves this by forcing a specific tool call with tool_choice. Both guarantee valid JSON.

6.2 The Agentic Loop ‚Äî Multi-Step Reasoning

Real agents need to call multiple tools, reason between steps, and decide when they're done.

flowchart TB Start["User Request"] --> Call["Send to Model"] Call --> Check{"stop_reason?"} Check -->|"stop / end_turn"| Done["Return Final Answer"] Check -->|"tool_calls / tool_use"| Exec["Execute Tool(s)"] Exec --> Append["Append Results to Messages"] Append --> Call style Start fill:#ede9fe,stroke:#8b5cf6,stroke-width:2px style Done fill:#dcfce7,stroke:#16a34a,stroke-width:2px style Check fill:#fef3c7,stroke:#f59e0b,stroke-width:2px style Exec fill:#fce7f3,stroke:#ec4899,stroke-width:2px

OpenAI Agentic Loop

// Pseudocode
while True:
  response = openai.chat.completions.create(
    model="gpt-4o",
    messages=messages,
    tools=tools
  )
  msg = response.choices[0].message
  messages.append(msg)

  if msg.tool_calls:
    for call in msg.tool_calls:
      result = execute(call.function)
      messages.append({
        "role": "tool",
        "tool_call_id": call.id,
        "content": result
      })
  else:
    break  # Done!

print(msg.content)

Anthropic Agentic Loop

// Pseudocode
while True:
  response = anthropic.messages.create(
    model="claude-sonnet-4-5",
    max_tokens=4096,
    messages=messages,
    tools=tools
  )
  messages.append({
    "role": "assistant",
    "content": response.content
  })

  tool_uses = [b for b in response.content
               if b.type == "tool_use"]
  if tool_uses:
    results = []
    for tu in tool_uses:
      result = execute(tu.name, tu.input)
      results.append({
        "type": "tool_result",
        "tool_use_id": tu.id,
        "content": result
      })
    messages.append({"role":"user",
                     "content": results})
  else:
    break  # Done!

print(response.content[0].text)

6.3 How Responses API Simplifies Agents

With built-in tools, the Responses API handles tool execution internally ‚Äî no agentic loop needed for built-in tools.

// One call does everything - model searches web, reads files, runs code
POST /v1/responses

{
  "model": "gpt-4o",
  "input": "Find the latest NVIDIA stock price, calculate the P/E ratio, and create a chart",
  "tools": [
    {"type": "web_search"},
    {"type": "code_interpreter"}
  ],
  "store": true
}

// The API orchestrates internally:
// 1. Searches web for stock price
// 2. Searches web for earnings data
// 3. Runs Python to calculate P/E ratio
// 4. Generates chart image
// 5. Returns combined result
// All in ONE API call!

When You Still Need an Agentic Loop

No loop needed (Responses API)

  • web_search, file_search, code_interpreter
  • image_generation
  • Any built-in tool

Loop still needed (both providers)

  • Custom functions (your code)
  • Database queries
  • External API calls
  • Any tool requiring your execution
