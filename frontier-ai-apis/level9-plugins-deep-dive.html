9.1 What Is a Plugin?

Anthropic ‚Äî A plugin is a file-based, shareable bundle that combines multiple capabilities into a single installable unit. Plugins are SDK-level ‚Äî they work in Claude Code and Cowork, not directly in the Messages API.

A plugin bundles:

  • Skills ‚Äî SKILL.md files (model-invoked domain expertise)
  • MCP connectors ‚Äî external tools via Model Context Protocol
  • Slash commands ‚Äî user-triggered workflows (e.g. /sales:prospect-research)
  • Sub-agents ‚Äî specialized agents for parallel workstreams
  • Hooks ‚Äî event handlers for pre/post processing

All defined in a .claude-plugin/plugin.json manifest.

OpenAI ‚Äî OpenAI does not have a "plugin" primitive in the same sense. The closest equivalent is the OpenAI Agents SDK, where you compose agents with tools + handoffs. OpenAI deprecated their old ChatGPT Plugins in 2024.

9.2 How a Plugin Uses the Messages API Under the Hood

Anthropic ‚Äî When the Claude Agent SDK loads a plugin, it orchestrates calls to the Messages API. Here is the flow:

Plugin loaded

SDK reads plugin.json, discovers skills, commands, and MCP server definitions.

MCP servers started

SDK starts MCP server processes, fetches available tool lists from each server.

Messages API call constructed

system prompt includes skill instructions from SKILL.md files. tools array includes MCP tools from connectors. messages contains the user prompt.

Model responds

If the response contains tool_use blocks, SDK routes each call to the correct MCP server.

Tool result returned

SDK sends tool_result back to the Messages API in a follow-up request.

Loop continues

The agentic loop repeats until the model returns a final text response with end_turn.

9.4 OpenAI: Agents SDK ‚Äî The Equivalent Architecture

OpenAI ‚Äî OpenAI's equivalent is not a "plugin" but the Agents SDK with composable agents, tools, and handoffs.

Core concepts:

  • Agent ‚Äî primary LLM with instructions + tools
  • Tools ‚Äî Python functions, built-ins (web_search, shell, code_interpreter), MCP servers
  • Handoffs ‚Äî agent-to-agent delegation (like sub-agents)
  • Agent-as-Tool ‚Äî one agent wraps another as a callable tool

Agents SDK example:

from openai_agents import Agent, handoff

triage_agent = Agent(
    name="Triage",
    instructions="Route customer requests",
    tools=[lookup_order],
    handoffs=[refund_agent, technical_agent]
)

refund_agent = Agent(
    name="Refund Specialist",
    instructions="Handle refund requests",
    tools=[process_refund, check_eligibility]
)

technical_agent = Agent(
    name="Tech Support",
    instructions="Resolve technical issues",
    tools=[search_kb, create_ticket]
)
