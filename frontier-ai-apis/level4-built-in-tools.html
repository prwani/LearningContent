4.1 OpenAI Responses API ‚Äî Built-in Tools

The Responses API has first-class support for built-in tools that OpenAI hosts and executes for you ‚Äî no function implementation needed.

web_search

Search the web in real-time

"tools": [
  {"type": "web_search"}
]

file_search

Search uploaded files/vector stores

"tools": [
  {"type": "file_search",
   "vector_store_ids": ["vs_..."]}
]

code_interpreter

Execute Python code in sandbox

"tools": [
  {"type": "code_interpreter"}
]

image_generation

Generate images inline

"tools": [
  {"type": "image_generation"}
]

mcp

Connect to external MCP servers

"tools": [
  {"type": "mcp",
   "server_url": "https://..."}
]

shell

Full Linux terminal in a container

"tools": [
  {"type": "shell",
   "environment":
     {"type": "container_auto"}}
]

computer_use

Control desktop UI via screenshots + clicks

"tools": [
  {"type": "computer_use_preview",
   "display_width": 1024,
   "display_height": 768,
   "environment":
     {"type": "browser"}}
]

function

Your custom functions (same as before)

"tools": [
  {"type": "function",
   "name": "...",
   "parameters": {...}}
]

Full Example ‚Äî Responses API with Web Search

POST /v1/responses

{
  "model": "gpt-4o",
  "input": "What are the latest AI regulations in the EU?",
  "tools": [{"type": "web_search"}],
  "instructions": "Cite your sources.",
  "store": true
}

// OpenAI executes the web search FOR you
// No function code needed on your side
// Results are woven into the response

4.2 Anthropic Messages API ‚Äî Built-in Tools

Anthropic also has built-in tools that run server-side or require your app to relay execution. Specified via versioned type identifiers.

web_search

Real-time web search (server-side)

"tools": [{
  "type": "web_search_20250305",
  "name": "web_search"
}]

bash

Shell command execution (client-side)

"tools": [{
  "type": "bash_20250124",
  "name": "bash"
}]

code_execution

Run code in sandbox (server-side)

"tools": [{
  "type": "code_execution_20250825",
  "name": "code_execution"
}]

computer

Desktop control via screenshots + clicks

"tools": [{
  "type": "computer_20250124",
  "name": "computer",
  "display_width_px": 1024,
  "display_height_px": 768
}]

web_fetch

Fetch specific URLs/APIs (server-side)

"tools": [{
  "type": "web_fetch_20250805",
  "name": "web_fetch"
}]
// No OpenAI equivalent

text_editor

File operations (client-side)

"tools": [{
  "type": "text_editor_20250124",
  "name": "str_replace_editor"
}]
Server vs Client: Tools like web_search, code_execution, and web_fetch run on Anthropic's servers. Tools like bash, computer, and text_editor require your app to execute commands and return results.

4.3 Advanced Tool Patterns

Tool Search

Anthropic Dynamic discovery across large tool libraries. Model searches for the right tool at runtime instead of listing 100+ tools in context.

Programmatic Tool Calling

Anthropic Claude orchestrates tools via code ‚Äî loops, conditionals, error handling ‚Äî all in one reasoning step.

MCP (Model Context Protocol)

Both Universal open protocol for connecting to external services. Supported by both OpenAI and Anthropic.

4.4 How Built-in Tools Work ‚Äî Flow Diagrams

OpenAI Responses API ‚Äî Built-in Tool Flow
sequenceDiagram participant U as User participant A as Your App participant R as Responses API participant T as OpenAI Hosted Tool U->>A: "Latest EU AI regulations?" A->>R: input + tools: web_search R->>T: Execute web_search internally T-->>R: Search results R->>R: Model synthesizes answer R-->>A: Final response with citations A-->>U: Display answer Note over A,R: One API call - OpenAI handles tool execution
Anthropic Computer Use Tool Flow
sequenceDiagram participant U as User participant A as Your App participant M as Messages API participant D as Desktop VM U->>A: "Fill out the expense form" A->>M: message + tools: computer M-->>A: tool_use: screenshot A->>D: Take screenshot D-->>A: Screenshot image A->>M: tool_result: image M-->>A: tool_use: click 450,320 A->>D: Click at coordinates A->>M: tool_result: done M-->>A: tool_use: type "Q1 Report" A->>D: Type text A->>M: tool_result: done M-->>A: "Expense form filled out" A-->>U: Done Note over A,M: Multiple round-trips - you control the VM
Key insight: OpenAI built-in tools (web_search, code_interpreter) execute server-side in one call. Computer use on both providers follows the multi-round-trip pattern ‚Äî your app must relay screenshots and execute UI actions.
